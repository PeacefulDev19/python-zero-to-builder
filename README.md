## 🤖 AI Usage Rules & Prompt Templates

### Golden Rules

| ✅ DO | ❌ DON'T |
|---|---|
| Ask AI to **explain** a concept you don't understand | Ask AI to **write code you don't understand** |
| Ask AI to **review** your code and suggest improvements | Ask AI to **write your project** for you |
| Ask AI to **generate test cases** for your functions | Copy-paste AI code without reading it |
| Ask AI to **compare approaches** (dict vs list for X) | Blindly trust AI-generated code |
| Use AI to **generate project ideas** when stuck | Use AI as a **replacement for thinking** |

### Prompt Templates to Copy-Paste

**Concept Explanation:**
```
Explain [CONCEPT] in Python like I'm 14. Use an analogy related to [video games / sports / something I like]. Give me 3 examples.
```

**Code Review:**
```
Here's my Python code for [PROJECT]. Review it for:
1. Bugs or potential errors
2. Ways to make it cleaner / more Pythonic
3. Features I could add next
Don't rewrite it — just suggest improvements.
```

**Debug Help:**
```
I'm getting this Python error:
[PASTE ERROR MESSAGE]
Here's the relevant code:
[PASTE CODE]
Explain what's wrong in simple terms, and give me a hint to fix it — don't give me the full solution.
```

**Project Ideas:**
```
I just learned about [TOPIC] in Python. Suggest 3 beginner-friendly project ideas that use [TOPIC]. Make them fun and something a teenager would actually want to use.
```

**Test Case Generation:**
```
I wrote this Python function:
[PASTE FUNCTION]
Generate 10 test cases (including edge cases) to verify it works correctly. Include expected inputs and outputs.
```

## ⚠️ Common Pitfalls & How to Avoid Them

### 1. Tutorial Hell
**Problem**: Watching videos but never building anything.
**Fix**: Watch 20 mins max, then code for 45 mins. If you're not building, you're not learning.

### 2. Copy-Paste Syndrome
**Problem**: Copying code from tutorials without understanding it.
**Fix**: Type every character yourself. If you must paste, delete it and retype it line by line.

### 3. AI Dependency
**Problem**: Asking AI to write everything, then not understanding your own code.
**Fix**: Follow the AI Usage Rules above. Ask for hints, not solutions.

### 4. Perfectionism
**Problem**: Your code isn't "clean" or "perfect" so you keep rewriting instead of moving forward.
**Fix**: "Done is better than perfect." Version 1 doesn't need to be good — it needs to exist.

### 5. Feature Creep
**Problem**: Adding too many features to your project, never finishing.
**Fix**: Define a **minimum viable project** before you start. Build that first, add features later.

### 6. Giving Up on Errors
**Problem**: Seeing a red error message and thinking you broke something.
**Fix**: Errors are normal. Every error is a learning moment. Read the error message, paste it into AI, and understand the fix.

### 7. Not Using Version Control
**Problem**: Losing work, not being able to go back.
**Fix**: `git commit` every day. Your GitHub history is your resume.

## 🎯 Key Principles for Success

1. **Code every single day** — even 20 minutes counts. Consistency > intensity.
2. **Type everything yourself** — never copy-paste. Muscle memory + understanding.
3. **Build 7+ real projects** by Day 30. These go in your portfolio.
4. **Use AI as a tutor** — it's your 1-on-1 teaching assistant, not your coder.
5. **Get comfortable with errors** — red text doesn't mean failure. Every error is a learning moment.
6. **Teach what you learn** — write notes, explain concepts, build things.
7. **Push to GitHub daily** — your commit history is your resume.
